> SearchGrid is a UI component designed to represent tabular row data. It enables filtering out results with quick querying.

Available Properties

name (Required): Name of the component. Default is SearchGrid.
title (Optional): SearchGrid title.
url (Required): The default URL to be used for requests to the server to retrieve the data.
rootTag (Required): The root node name of AJAX response which SearchGrid takes as starting point to get its data. ex: "Objects" for the sample data below
contentPath (Required): The node names of AJAX response which will be used as row data. ex: "Object" for the sample data below
keyAttribute (Required): The attribute name of the row node which uniquely identifies the node. ex: "id" for the sample data below
defaultSearchClass (Required): Default class that the search will be applied.
fieldsUrl (Required): The url used for the request to the server to retrieve available properties used in view builder.
pollingInterval (Optional): Time delay between two server requests. Default is 0 which means there is no polling.
queryParameter (Required): The url parameter to send the query to the server. The value of this property depends on your implementation for the server side code implemented for the "url" property above.
searchInEnabled (Required): Determines if the query should be applied on only defaultSearchClass or should be selected among classes which are brought by searchClassesUrl.

All properties except rootTag, contentPath and keyAttribute are pre-populated when a new SearchGrid component is added to the layout, assuming you will be using SearchController provided with RapidInsight.

Additional Properties

queryEnabled (Optional): Parameter to determine whether the quick filtering is enabled or not. Default is true.
bringAllProperties (Optional): When set to false SearchGrid requests its data with a parameter ("propertyList") to indicate that it needs only a set of properties, to decrease the size of the data coming from server.
extraPropertiesToRequest (Optional): Comma separated property names which will be added "propertyList" URL parameter. Active when bringAllProperties is set to false.
multipleFieldSorting (Optional): When set true component supports multiple field sorting.
searchClassesUrl (Required when queryEnabled is "true"): The url used for the request to the server to retrieve available search classes.
maxRowsDisplayed (Optional): The maximum row count requested from server at every poll. Default is 100.
defaultQuery (Optional): The query appended to all queries sent by SearchGrid.
timeout (Optional): The time interval in seconds to wait the server request completes successfully before aborting.

Available Actions

Clone: Creates a copy of the component with same properties
Delete: Deletes the component

Available Events

onSaveQueryClick: Event fired when "Save Query" button is clicked. Available parameters:

  • params.query: Query written in search input field.

onRowDoubleClick: Event fired when a row is double clicked. Available parameters:

  • params.data: JavaScript object representing row data. While defining an action on this event, the row data can be accessed through params.data object. ex: params.data.severity, params.data.id, etc.

onRowClick: Event fired when a row is clicked. Available parameters:

  • params.data: JavaScript object representing row data. While defining an action on this event, the row data can be accessed through params.data object. ex: params.data.severity, params.data.id, etc.
