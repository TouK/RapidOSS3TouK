A TreeGrid component is used to display the data in hierarchical form. You can have both a tree structure with expanding/collapsing nodes as well as a grid view attached to the expanding nodes as follows:

Available Properties

url (Required): The default URL to be used for requests to the server to retrieve hierarchical data. ex:
rootTag (Required): The root node name of AJAX response which TreeGrid takes as starting point to get its data. ex: "Objects" for the sample data below
contentPath (Required): The node names of AJAX response which will be used as row data. ex: "Object" for the sample data below
keyAttribute (Required): The attribute name of the row node which uniquely identifies the node. ex: "id" for the sample data below
pollingInterval (Optional): Time delay between two server requests. Default is 0 which means there is no polling.
name (Required): Default is TreeGrid. The name does not have to be unique.
title (Optional): TreeGrid title.

Below is the sample Ajax response in Xml format to a url call:

<Objects>
   <Object id="London" name="London" nodeType="Container" >
	<Object id="AM" name="AM" displayName="AM" nodeType="Container" state="1">
	    <Object id="51" name="ercpsvsyr1" displayName="ercpsvsyr1" nodeType="Object" rsDatasource="AM"  state="1"/>
	    <Object id="56" name="ercwsvnyc1" displayName="ercwsvnyc1" nodeType="Object" rsDatasource="AM"  state="1"/>
	</Object>
   </Object>
   <Object id="Paris" name="Paris" nodeType="Container">
	<Object id="SAM" name="SAM" displayName="SAM" nodeType="Container"  state="4">
	    <Object id="52" name="erctsvnyc8" displayName="erctsvnyc8" nodeType="Object" rsDatasource="SAM"  state="4"/>
	    <Object id="53" name="ercpsvroc" displayName="ercpsvroc" nodeType="Object" rsDatasource="SAM"  state="3"/>
	</Object>
   </Object>
</Objects>

Additional Properties

tooltip (Optional): Parameter to display a tooltip over rows. Default is false.
expanded (Optional): Parameter to display TreeGrid branches either expanded or collapsed. Default is false.

Available Actions

Delete: Deletes the component

Events

onNodeClick: Event fired when a row is clicked. Available parameters:

  • params.data: JavaScript object representing the row data. While defining an action on this event, the row data can be accessed through params.data object. ex: params.data.state, params.data.id, etc.

Child Nodes

params.data is available for the visible parameter to write expressions to determine when individual child nodes such as images, menu items are to be displayed.
Columns

A TreeGrid component must have at least one TreeGridColumn. A TreeGridColumn can be either text or an image. Default is text.

  • attributeName (Required): The data node attribute which will be shown in the column. This attribute comes with the Ajax response. ex: "displayName" for the sample data above.
  • colLabel (Required): Title of the column.
  • width (Required): Width of the column.
    Additional Properties:
  • sortBy (Optional): Parameter to render TreeGrid whether sorted according to this column or not. Default is false.

If a TreeGridColumn is of type Image, you can add one or more images under Images node. The display of the image depends on an expression defined in "visible" parameter.

  • src (Required): Image url. ex: images/rapidjs/component/states/green.png
  • visible (Optional): The JavaScript expression evaluated on row data to determine whether the image is displayed or not. Default is "true" which means it is always displayed. ex: params.data.state =='0'
    Additional Properties:
  • align (Optional): Sets the alignment of the image. Available values are "left", 'right" and "center". Default is "left".
Root images

These are the images that will be displayed on the left side of each data row in the TreeGrid depending on the row being expanded or collapsed.

  • expanded (Required): The image url which will be shown when the row is expanded. ex: images/rapidjs/component/tools/folder_open.gif
  • collapsed (Required): The image url which will be shown when the row is collapsed. ex: images/rapidjs/component/tools/folder.gif
  • visible (Optional): The JavaScript expression evaluated on row data to determine whether the image is displayed or not. Default is "true". params.data object will be used to write the expression. ex: params.data.nodeType=='Container'
Menu items

They are data row menu items.

  • name (Required): The name of the menu item
  • label (Required): The label of the menu item.
  • visible (Optional): The JavaScript expression evaluated on row data to determine whether the item is displayed or not. Default is "true". params.data object will be used to write the expression. ex: params.data.nodeType=='Container'
Note that the actions which will be executed when a menu item is selected are defined in Actions node for the tab.
