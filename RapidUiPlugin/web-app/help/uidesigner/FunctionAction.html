When triggered, this action will invoke a function of another UI component.

Available Properties

name: Name your function action
component: Select the component whose function will be invoked
function: Select the function of the component. List will be populated with the available functions of the selected component.

All components provide below functions:

  • poll
  • refresh
  • setTitle(argument list: title)

SearchList component provides these additional functions:

  • setQuery(argument list: queryString, sortAttributes, sortOrder, searchIn, extraParams)
  • appendToQuery(argument list: query)
  • appendExceptQuery(argument list: property, value)
  • sort(argument list: sortAttribute, sortOrder)

SearchGrid component provides these additional functions:

  • setQueryWithView(argument list: queryString, view, searchIn, title, extraParams)
  • appendToQuery(argument list: query)
  • appendExceptQuery(argument list: property, value)
  • sort(argument list: sortAttribute, sortOrder)

Html component provides these additional functions:

  • show(argument list: url, title)

"url" can be provided using the createURL utility method. 

ex:createURL('getEventDetails.gsp', {name:params.data.name})

ObjectMap component provides two additional functions:

  • loadMap(argument list: response)

This function loads the map from AJAX response. "response" is the AJAX response object which includes map structure, ie: mapPropertyList , mapProperties , nodePropertyList, layout.

  • loadMapForNode (argument list: nodeParams, mapParams)
    This function loads the map for the given object name.
    nodeParams is the map of properties that will identify the root node of the map. 
    ex: {name:'sampleNode',rsClassName:'RsTopologyObject' }

    mapParams: Map of properties that will identify the map.

    ex : { mapType:'sampleMap' }

The properties specified in nodePropertyList and mapPropertyList properties of ObjectMap component will be searched from nodeParams and mapParams.

condition: Specify a condition if you want this action to be executed conditionally.

First, define your triggers and then come back and edit the condition since you will need to know what data is made available by the fired event selected by the trigger.

For example, you have defined a trigger for the SearchGrid component's propertyClicked event. But the requirement is to execute the action only when the classname column is clicked. propertyClicked event of SearchGrid makes the following data available:

  • params.data
  • params.key
  • params.value
    This is what we need to write as the condition Javascript expression: 
    params.key == 'className'

    The action will be executed only when this expression return true.

function arguments: Based on the selected function, the arguments required by the function will be added to the property list of the action.

First, define your triggers and then come back and edit the condition since you will need to know what data is made available by the fired event selected by the trigger.

View the documentation pane to see what data is made available by the defined triggers. A javascript expression is written as the value of the argument.
For example, when the setQueryWithView function of the SearchGrid is selected, following 4 additional properties will be added to the Property Editor, each corresponding to an argument of the setQueryWithView function:

  • arg0:queryString
  • arg1:view
  • arg2:title
  • arg3:extraParams

Available Actions

Clone: Clones the action to easily create another similar action
Delete: Deletes the action
