Actions node is the parent node for all actions defined for the selected tab. Actions govern the interactions between components.

Multiple UI components communicate through events fired and consumed. For example, selecting a node on a TreeGrid component fires an event that can be consumed by another component. Another example would the firing of an event which triggers an action targeting the same component or another component. What triggers an action, what is the action logic, and what component is impacted by the action are all configured through the Action Configuration dialog.

Actions node has no properties of its own.

Available Actions

Add Action: Adds a new Action.

In order to guide the user while defining an action, an action configuration dialog will pop-up to collect information, before the node for the action is added. Once the Action is added its additional properties can be set in the property editor. The action configuration dialog can be re-opened to edit the configuration.

Action Configuration Key Concepts

Type

RapidInsight UI components support various actions such as making a server-side call on a node clicked event, making a function call to another UI component based on a menu click, etc. To support these different behaviors, multiple action types are supported:
request: When triggered, this action will make a server side call.
merge: When triggered, this action will make a server side call (specified by the url field) and update its content (merge the data it has with the data returned from the server).
link:
function: When triggered, this action will invoke a function/method of another UI component. The component and its method will also be specified in this dialog.
combined: When triggered, multiple actions can be executed.

Trigger Type

Designates what triggers the action; an event, such as a node-clicked event, or a menu, such as "Details" menu on a device.
If the trigger type is event, a component and one of its supported events is added to the Triggering Events list. More than one "component/its supported event" pair can be added to the list.

Specify what the Action should do

Depending on the type of the action, this means specifying:

  • a target component and its method
  • the url of the server-side call
Action dialog will show what data will be available to be passed around when an event is triggered, or when a menu is selected. Similarly, it will show what arguments are expected if the action will invoke a method of another UI component
