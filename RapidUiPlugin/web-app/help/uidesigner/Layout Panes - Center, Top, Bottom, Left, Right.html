> Layout panes are where UI components are placed. A sub layout can be added to each pane for more creating complex layouts.

Available Properties

component: (Optional) Select from a drop down of components defined under the Components node.
height: (Required for Top and Bottom panes) The height (in pixels) that the unit will take up in the layout.
width: (Required for Left and Right panes) The width (in pixels) that the unit will take up in the layout.
resize: (Optional) Boolean indicating whether this unit is resizeable. Not available in Center pane.

Additional Properties

contentFile: (Optional) Content file where the layout content will be retrieved
gutter: (Optional) The gutter applied to the unit's wrapper, before the content.
scroll: (Optional) Boolean indicating whether the unit's body should have scroll bars if the body content is larger than the display area.
useShim: (Optional) This setting will be passed to the DragDrop instances on the resize handles and for the draggable property. This property should be used if you want the resize handles to work over iframe and other elements.
minWidth: (Optional) The minimum width of unit when resizable property is set to "true". Available in Right and Left panes.
maxWidth: (Optional) The maximum width of unit when resizable property is set to "true". Available in Right and Left panes.
minHeight: (Optional) The minimum height of unit when resizable property is set to "true". Available in Top and Bottom panes.
maxHeight: (Optional) The maximum height of unit when resizable property is set to "true". Available in Top and Bottom panes.
collapse: (Optional) Boolean incidcationg whether the unit is collapsible or not. Available in Right, Left, Top and Bottom panes.

Available Actions

Add Layout: Adds a sub layout node to further divide the pane
Clone: Creates a copy of the layout with the same properties
Delete: Deletes the pane from the layout (Note: Center pane CANNOT be deleted)
