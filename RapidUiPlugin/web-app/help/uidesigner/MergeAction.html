> When triggered, this action will make a server side call (specified by the url field) and update its own data or the data of another component (merge the data it has with the data returned from the server).

Available Properties

name: Name your merge action
url: Enter the URL of the script to be invoked (start with scripts folder, i.e. scripts/acknowledge.groovy)
timeout: Timeout specifies, in seconds, when the call to the server will be considered to have timed out.
components: Comma separated list of component names whose data will be updated/merged with the data returned from the server.
submitType: HTTP method used to send the request to the server. Possible values are "GET" and "POST".
condition: Specify a condition if you want this action to be executed conditionally.

First, define your triggers and then come back and edit the condition since you will need to know what data is made available by the fired event selected by the trigger.

For example, you have defined a trigger for the SearchGrid component's propertyCliecked event. But the requirement is to execute the action only when the classname column is clicked. propertyCliecked event of SearchGrid makes the following data available:

  • params.data
  • params.key
  • params.value
    This is what we need to write as the condition Javascript expression: 
    params.key == 'className'

    The action will be executed only when this expression return true.

Additional Properties

removeAttribute: When the result returned from the server contains the attribute specified here for a particular row, the corresponding row will be removed from the component's data. 

Available Actions

Clone: Clones the action to easily create another similar action
Delete: Deletes the action

The request parameters that should be passed to the server-side will be defined by adding one or more RequestParameters.

Make sure that you have defined your triggers before adding request parameters since the triggers will determine what data will be available for passing into the request parameter.
